# Data Dependencies

This project expects NFL Big Data Bowl 2025 CSV files to be available locally. The Makefile assumes they are in the repository root by default, but every command can use another location with `DATA_DIR=/path/to/data`.

## Required Files

The main expected receiver yards pipeline requires:

- `plays.csv`
- `player_play.csv`
- `players.csv`
- `tracking_week_1.csv` through `tracking_week_9.csv`, or whichever tracking weeks are selected with `--weeks`

The analysis pipeline requires the same metadata files plus the output files generated by `expected_receiver_yards.py`.

The graph transformer prototype also requires:

- `plays.csv`
- `player_play.csv`
- `players.csv`
- selected `tracking_week_*.csv` files

## Core Input Columns

`plays.csv` is used for game context, pass outcome, coverage, EPA, down, distance, and time-to-throw fields. Important columns include:

- `gameId`
- `playId`
- `possessionTeam`
- `defensiveTeam`
- `down`
- `yardsToGo`
- `preSnapHomeScore`
- `preSnapVisitorScore`
- `absoluteYardlineNumber`
- `passResult`
- `timeToThrow`
- `expectedPointsAdded`
- `yardsGained`
- `pff_passCoverage`
- `pff_manZone`

`player_play.csv` is used to identify passers, route runners, targets, sacks, receptions, interceptions, and receiver production. Important columns include:

- `gameId`
- `playId`
- `nflId`
- `hadDropback`
- `wasRunningRoute`
- `wasTargettedReceiver`
- `hadPassReception`
- `wasSacked`
- `causedPressure`
- `routeRan`
- `receivingYards`

`players.csv` is used for player names and positions. Important columns include:

- `nflId`
- `displayName`
- `position`

`tracking_week_*.csv` is used for frame-level player locations, movement, team, event, and play direction. Important columns include:

- `gameId`
- `playId`
- `nflId`
- `displayName`
- `frameId`
- `club`
- `playDirection`
- `x`
- `y`
- `s`
- `a`
- `dir`
- `event`

## Local Data Policy

Raw tracking files are large and should not be committed to git. The included `.gitignore` excludes raw data, generated CSV outputs, trained model checkpoints, and local cache files.
